1. INTRODUCTION                                             *pkm-introduction*

PKM.nvim is a personal knowledge management plugin for Neovim. It manages
a collection of markdown notes with YAML frontmatter, automatic bidirectional
citation tracking, and Telescope integration for search and navigation.

The system is built around a three-stage workflow:

  Scratchpad  →  quick capture, no friction
      ↓
  Journal     →  timestamped entries, personal log
      ↓
  Consolidated →  permanent numbered knowledge base

Notes are plain markdown files. No proprietary formats, no cloud dependency.
Everything works from the filesystem. All notes share a single flat namespace
and a single sequential counter — note numbers are globally unique.

==============================================================================
2. REQUIREMENTS                                             *pkm-requirements*

  • Neovim ≥ 0.10
  • nvim-telescope/telescope.nvim   (optional but strongly recommended)
  • ripgrep (rg)                    (required for :PKMSearch)

==============================================================================
3. INSTALLATION                                             *pkm-installation*

lazy.nvim: >
  {
    'yourusername/pkm.nvim',
    dependencies = { 'nvim-telescope/telescope.nvim' },
    config = function()
      require('pkm').setup({
        root_path = vim.fn.expand('~/Notes'),
      })
    end,
  }
<

==============================================================================
4. SETUP                                                           *pkm-setup*

Minimal: >
  require('pkm').setup({
    root_path = vim.fn.expand('~/Notes'),
  })
<

With user info and custom keymaps: >
  require('pkm').setup({
    root_path = vim.fn.expand('~/Notes'),
    user = {
      name = "Your Name",
    },
    keymaps = {
      new_note        = "<leader>nn",
      new_journal     = "<leader>nj",
      search          = "<leader>nf",
      insert_citation = "<leader>nc",
    },
  })
<

PKM creates the folder structure under root_path on first use.
Set any keymap value to `false` to disable it.

5. WORKFLOW                                                     *pkm-workflow*

CAPTURE~
Use |:PKMNewScratchpad| to write down ideas immediately.
Scratchpads are timestamped files in 01-Scratchpad/. They carry minimal
frontmatter and require no decisions at creation time.

JOURNAL~
Use |:PKMNewJournal| for dated entries. The timestamp is recorded in the
filename and frontmatter automatically.

CONSOLIDATE~
When a scratchpad contains something worth keeping permanently, use
|:PKMPromote| to move it to a consolidated note or journal entry. You can
also create consolidated notes directly with |:PKMNewNote|.

CONSOLIDATED NOTE TYPES~
  note   Regular knowledge note
  bib    Bibliography or reference entry (prompts for author and source type)
  agg    Aggregate or collection note (gathers related content)

CITE~
In any note, use |:PKMInsertCitation| to pick another note and insert a
structured citation. The cited note automatically receives a backlink in its
`cited_by` frontmatter field. This is maintained bidirectionally.

SEARCH~
Use |:PKMSearch| to search note contents (requires ripgrep), or
|:PKMTags| to browse by tag.

6. COMMANDS                                                     *pkm-commands*

CREATION~

:PKMNewNote [type]      Create a consolidated note.
                        Type: 'note', 'bib', or 'agg'.
                        If omitted, a picker prompts for the type.

:PKMNewJournal          Create a journal entry timestamped to the current
                        time. Opens immediately for editing.

:PKMNewScratchpad       Create a new scratchpad with an optional title.


:PKMImport              Import the currently open file into the PKM structure.
                        Adds the appropriate frontmatter template for its
                        folder without overwriting existing fields.

LIFECYCLE~

:PKMConvertNote         Normalize the current note to match its folder format.
                        Adds any missing frontmatter fields while preserving
                        existing values. For consolidated notes without a
                        valid PKM filename, prompts for type and title and
                        renames the file.

:PKMPromote             Promote a scratchpad to a consolidated note or journal
                        entry. Only available from the scratchpad folder.

:PKMTranspose           Move the current note to a different PKM folder and
                        convert it to the appropriate type for that folder.
                        Useful when a note has outgrown its current context.

:PKMChangeType          Change the type of the current consolidated note
                        between 'note', 'agg', and 'bib'. Renames the file
                        and updates the frontmatter template fields.

:PKMDeleteNote          Safely delete the current note. Scans all other notes
                        for references to this file and removes them before
                        deletion. Always use this instead of deleting files
                        manually.

NAVIGATION~

:PKMFollowLink          Open the note referenced by the [[wiki-link]] under
                        the cursor. Searches all PKM folders.

:PKMLinkNote            Pick a note and insert a [[wiki-link]] at the cursor.

:PKMBacklinks           Show all notes that contain a link to the current note.

:PKMGotoCitation        Jump to the note referenced by the citation string
                        under the cursor (e.g. note[0042]).

SEARCH & BROWSE~

:PKMSearch              Raw streaming text search via Telescope live_grep
                        and ripgrep. Requires ripgrep. Scoped to note body
                        content only; use :PKMBrowse for structured queries.

:PKMBrowse [expr]       Browse PKM notes with an optional filter expression.
                        Supports tag:, title:, text:, and filename: predicates
                        combined with AND, OR, NOT, and parentheses. Examples:
                          :PKMBrowse
                          :PKMBrowse tag:math AND title:fourier
                          :PKMBrowse filename:0042
                        Uses the Telescope picker when available, vim.ui.select
                        otherwise. The prompt applies exact substring narrowing
                        over the pre-filtered set.

:PKMTags                Browse notes by tag. Select a tag from the full tag
                        list, then :PKMBrowse opens pre-filtered to that tag.
                        Uses the index — no ripgrep required.

:PKMMergeTags           Merge one or more tags into a target tag across all
                        notes. Three-step Telescope picker (target → sources →
                        confirm), or a fallback UI without Telescope.

:PKMStats               Show note counts per folder via vim.notify.

LIFECYCLE~

:PKMRenameNote          Prompt for a new filename (pre-filled with the current
                        stem). Sanitizes the input, renames the file on disk,
                        redirects the buffer, and propagates the rename through
                        all citations. Does not modify the title field.

CITATIONS~

:PKMInsertCitation      Open a picker of all citable notes and bibliography
                        entries. Inserting a selection writes the citation
                        string and updates both notes' frontmatter.

:PKMGotoCitation        Jump to the note under the cursor.

:PKMUpdateReferences    Scan the current buffer for citation strings and
                        rebuild the `cites` frontmatter list to match.

EXPORT~

:PKMExport              Open the export filter form. See |pkm-export|.

VIEWS~

:PKMView [name]         Open a named project view. Without a name, presents a
                        picker of all defined views. Tab-completes view names.
                        Simple views are filter expressions (strings). Subproject
                        views are defined in views.json with parent and filter
                        fields; their effective filter is the parent's AND own.

:PKMViews               Open a tree-structured picker of all defined views,
                        showing the parent-child hierarchy with note counts.
                        Root views appear first; subprojects are indented under
                        their parent. Select a view to open it.
                        Example display:
                          • rpg  (8)
                          ▶ ringforge  (14)
                            ▶ ringforge-mechanics  (6)
                            • ringforge-references  (8)
                        Uses Telescope when available; scrollable float
                        otherwise.

:PKMViewNew             Prompt for a name and filter expression, then save to
                        views.json. For subprojects, use :PKMViewEdit directly.

:PKMViewEdit            Open views.json for direct editing. Saving the file
                        automatically reloads all views. Use this to define
                        subprojects:
                          {
                            "physics": "tag:physics",
                            "physics-mechanics": {
                              "parent": "physics",
                              "filter": "tag:mechanics"
                            }
                          }

:PKMViewDelete          Remove a named view from views.json. Tab-completes
                        view names.

:PKMViewLast            Reopen the last view activated in this session.
                        Session-scoped; does not persist across restarts.

:PKMViewSidebar [name]  Open or toggle the persistent sidebar for a named view.
                        Without a name, prompts for one (or closes if open).
                        Same name again → closes; different name → replaces.
                        When the view participates in the subproject hierarchy,
                        a tree header is shown above the note list:
                          ▶ parent-view  (N notes)
                          ▼ current-view (N notes)
                          ▶ child-view   (N notes)
                          ──────────────────────
                          ...note list...
                        Sidebar keymaps:
                          <CR>   open note in last focused window (on note line)
                                 or navigate to that view (on tree header line)
                          <BS>   navigate to parent view
                          r      refresh against current index
                          q / <Esc>   close sidebar
:PKMExportView [name]   Export all notes in a named view, skipping the
                        interactive filter form. Without a name, presents a
                        picker. Tab-completes view names. Opens the results
                        picker then prompts for a destination directory.

BUFFER PANEL                                                *pkm-bufpanel*

:PKMBuffers             Toggle the persistent bottom buffer-list panel.
                        Lists all open regular-file buffers. PKM notes show
                        their type prefix and title; non-PKM files show their
                        filename. Modified buffers are marked [+].

                        Keymaps inside the panel:
                          <CR>   open buffer in the main editing window
                          d      close buffer (:bdelete)
                          D      force-close buffer (:bdelete!)
                          w      write buffer then close it
                          r      refresh the list
                          q      close the panel

==============================================================================
MARKDOWN EDITING~

:PKMNextHeader          Duplicate the header on the current line with its
                        trailing counter incremented, append at EOF.

:PKMHeaderLevelUp       Increase the # depth of all headers in the range.
                        Default range: whole buffer. Also accepts '<,'> prefix
                        for visual selection.

:PKMHeaderLevelDown     Decrease the # depth of all headers in the range.
                        Level-1 headers are protected.

:PKMHeadingNext         Jump to the next ATX heading (# …) in the buffer.

:PKMHeadingPrev         Jump to the previous ATX heading in the buffer.

7. CITATIONS                                                   *pkm-citations*

CITATION FORMAT~

Citations are written inline as: >
  note[0042]
  bib[0003]
<
The number is the four-digit note number. The prefix identifies the note type.

INSERTING A CITATION~

Use |:PKMInsertCitation| (or the keymap). A Telescope picker shows all notes
with their identifiers and titles. Selecting one inserts the citation string
at the cursor and immediately:

  1. Adds the cited note to the current file's `cites.notes` or `cites.bib`
  2. Adds the current file to the cited note's `cited_by.notes` or `cited_by.bib`

BIDIRECTIONAL SYNC~

The frontmatter fields `cites` and `cited_by` are always kept in sync:

  • When you insert a citation: both files are updated immediately
  • On save (if auto_sync_on_save = true): :PKMUpdateReferences rescans
    the buffer and repairs any inconsistencies
  • When you delete a note: :PKMDeleteNote removes all references to it

FRONTMATTER STRUCTURE~
>
  cites:
    notes:
      - identifier: note-0042
        title: "Note Title"
        link: "[[0042_note_Note_Title]]"
    bib:
      - identifier: bib-0003
        title: "Source Title"
        link: "[[0003_bib_Source_Title]]"
  cited_by:
    notes: []
    bib: []
<
Empty groups serialize as `[]` (not omitted). Do not change this structure
manually; use the PKM commands instead.

8. EXPORT                                                         *pkm-export*

:PKMExport opens a filter form to select notes and copy them to a folder.
Useful for preparing a set of notes to review, share with an LLM, or archive.

THE FILTER FORM~
>
  ╭─ PKMExport: Advanced Filter ──────────────────────────────────────╮
  │  Fill in any fields. Leave blank to skip.                         │
  │  Comma-separated values: OR logic (AND for the second tags field).│
  │  <Tab>/<S-Tab> move fields   <CR> search   <Esc> cancel           │
  │  ──────────────────────────────────────────────────────────────── │
  │  Tags ANY  (OR) :                                                 │
  │  Tags ALL (AND) :                                                 │
  │  Title contains :                                                 │
  │  Text  contains :                                                 │
  ╰───────────────────────────────────────────────────────────────────╯
<
  Tags ANY (OR)   Note has at least one of these tags.
                  Comma-separated. Example: mathematics, physics
  Tags ALL (AND)  Note has every one of these tags.
                  Comma-separated. Example: proof, analysis
  Title contains  Exact case-insensitive substring of the note title.
  Text  contains  Exact substring anywhere in the note body.

All active fields are AND-ed together. Leave any field blank to skip it.
Matching is exact substring — never fuzzy.

RESULTS PICKER~

After pressing <CR>, matched notes open in a Telescope picker (or a
scrollable float if Telescope is unavailable). Press <CR> to confirm the
selection, then enter a destination folder path.

PROGRAMMATIC API~
>
  -- Export all notes tagged both "mathematics" AND "proof"
  require('pkm.export').export(
    { tags_all = {"mathematics", "proof"} },
    "/tmp/output"
  )

  -- Tags: any of math/physics, AND body contains "Fourier"
  require('pkm.export').export(
    { tags_any = {"mathematics", "physics"}, text = "Fourier" },
    "/tmp/output"
  )
<

9. CONFIGURATION                                           *pkm-configuration*
>
  require('pkm').setup({
    -- Path to notes root directory. Required.
    -- Defaults to ~/Notes if omitted.
    root_path = vim.fn.expand('~/Notes'),

    folders = {
      scratchpad   = "01-Scratchpad",
      journal      = "02-Journal",
      consolidated = "03-Consolidated",
      templates    = "templates",
    },

    sync = {
      -- Master switch for all sync features
      enabled = true,
      -- Update last_updated_on and citation lists on every save
      auto_sync_on_save = true,
    },

    timestamp = {
      -- "full"      → YYYY-MM-DD_HH-MM-SS
      -- "date_time" → YYYY-MM-DD_HH-MM
      -- "date_only" → YYYY-MM-DD
      default_format = "full",
      auto_timestamp = true,
    },

    user = {
      name        = "",   -- Inserted in 'author' frontmatter field
      email       = "",
      institution = "",
    },

  -- Named project views. Each key is a view name; each value is a filter
  expression string. Activated with :PKMView <name>.
  Example:
    projects = {
      "rpg":    "tag:rpg AND (title:ringforge OR text:ringforge)",
      "ringforge-mechanics": {
        "parent": "ringforge",
        "filter": "tag:mechanics"
      }
      clinic = 'tag:medicine AND tag:protocol AND NOT tag:draft',
    }

  -- Width of the :PKMViewSidebar window (default 40 columns)
    sidebar_width = 40,

    -- Buffer-local symbol abbreviations and insert-mode keymaps.
    -- Registered per PKM note buffer via BufReadPost.
    -- trigger (iabbrev) and key (insert-mode map) are both optional.
    symbols = {
      { trigger = 'emdash', key = '<M-->', expansion = '—' },
      { trigger = 'sect',   key = '<M-s>', expansion = '§' },
      { trigger = 'ordm',   key = '<M-o>', expansion = 'º' },
    },

    keymaps = {
      -- Set any value to false to disable that keymap
      new_note         = "<leader>nn",
      new_journal      = "<leader>nj",
      new_scratchpad   = "<leader>ns",
      convert_note     = "<leader>nx",
      promote_note     = "<leader>np",
      transpose_note   = "<leader>nT",
      change_note_type = "<leader>nC",
      import_note      = "<leader>ni",
      delete_note      = "<leader>nd",
      insert_citation  = "<leader>nc",
      goto_citation    = "<leader>ng",
      link_note        = "<leader>nl",
      follow_link      = "gf",
      backlinks        = "<leader>nb",
      search           = "<leader>nf",
      browse_tags      = "<leader>nt",
      rename_note      = "<leader>nr",
      browse           = false,       -- :PKMBrowse (direct)
      view_last        = "<leader>nV",
      view_sidebar     = false,
      next_header      = "<leader>mh",
      header_level_up  = false,
      header_level_down = false,
      heading_next     = false,
      heading_prev     = false,
    },
  })
<

11. FRONTMATTER                                               *pkm-frontmatter*

CONSOLIDATED NOTE~

The `title` field is free-form. The system sets it at creation time from user
input but never overwrites it afterward. To rename the file, use
:PKMRenameNote — this changes the filename on disk without touching the title
field.
>
  ---
  title: ""
  author: ""
  created_on: "YYYY-MM-DD_HH-MM-SS"
  last_updated_on: "YYYY-MM-DD_HH-MM-SS"
  tags: []
  cites:
    notes: []
    bib: []
  cited_by:
    notes: []
    bib: []
  ---
<

12. FILE NAMING                                                  *pkm-naming*

CONSOLIDATED NOTES~

  NNNN_type_Title_Words.md

  NNNN   Four-digit zero-padded sequential number (globally unique)
  type   One of: note, bib, agg
  Title  Words from the title, joined by underscores, sanitized

Examples: >
  0042_note_Introduction_to_Fourier_Analysis.md
  0003_bib_Fourier_Analysis_Stein_Shakarchi.md
  0007_agg_Real_Analysis_Reading_List.md

To rename a consolidated note after creation, use |:PKMRenameNote|. It prompts
for a new name, sanitizes it, renames the file on disk, updates the buffer
reference, and propagates the change through all citations. The `title`
frontmatter field is never touched by the rename operation.

JOURNAL ENTRIES~

  journal_YYYY-MM-DD_HH-MM-SS.md
  (or shorter formats depending on timestamp.default_format)

SCRATCHPADS~

  scratch_YYYY-MM-DD_HH-MM-SS.md
